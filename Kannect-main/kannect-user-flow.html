KANNECT User Journey

Welcome to KANNECT—an intuitive platform fostering effective communication between students and faculty. This page guides you through the user flow, highlighting key steps, role-based experiences, core dashboard features, messaging, accessibility & security guarantees.

User Landing Overview

  • New and returning users arrive at the KANNECT homepage via direct link or invite.
  • Clear call-to-action buttons guide users to Register or Login.
  • Quick highlights on platform benefits and guidance for account creation.

User Onboarding & Account Creation

  1. User selects 'Register' and chooses their role (Student or Faculty).
  2. Simple registration form collects essential details: name, email, role, and affiliation.
  3. Platform presents accessible instructions at every step for a smooth onboarding experience.

New User Registration Process

Step 1

Data Collection

Basic information is gathered for account setup and role verification (name, email, role, institute).

Step 2

Email Verification

User receives a secure email with a verification link to confirm their identity and activate their account.

Step 3

Role Assignment

Platform validates the chosen role (Student or Faculty) and applies suitable access controls automatically.

Step 4

Account Creation

Successful verification grants access; user is guided to personalized dashboard based on their role.

Role-Based Access Control

Student

  • Access relevant learning resources and communication channels.
  • Dashboard displays assignments, notifications, and direct faculty messaging.

Faculty

  • Access course management, analytics, and messaging tools tailored for faculty tasks.
  • Can send announcements, provide feedback, and initiate discussions.

Note: Role assignment safeguards privacy and maintains secure boundaries between user groups.

User Authentication Flow

  1. User logs in with verified email and password.
  2. Credentials are securely validated. Multi-factor authentication (if enabled) for enhanced security.
  3. Upon success, user is redirected to their personalized dashboard based on assigned role.
  4. Session control prevents unauthorized access and supports secure log out.

Dashboard Features

Content & Guidance

Personalized modules, assignments, resources, and guidance appear upfront for each user.

Communication Tools

Integrated chat, announcements, and feedback systems facilitate timely exchange.

  • Simple menus and content cards ensure easy navigation.
  • Real-time notifications help users stay updated.

Messaging & Notification System

  • Real-Time Updates: Users receive instant alerts about assignments, messages, and platform activity.
  • Role-Based Messaging: Students connect directly to faculty; faculty can initiate group or personal messages.
  • Secure, easy-to-use chat interface supports attachments and questions.

Accessibility & Security Features

  • WCAG-compliant design—clear contrast, scalable text, keyboard navigation, and screen-reader compatibility.
  • Data encryption, secure APIs, and robust authentication protocols safeguard all user information.
  • Privacy-first approach: users manage their own profile and notification settings.

Seamless User Experience

  • Intuitive navigation—logical menu structure and clear section headings.
  • Minimalist interface—subtle colors and simple layouts keep focus on content.
  • Immediate feedback for every user action (registration, login, messaging).
  • All flows use plain language and context-sensitive help for clarity.
